# 什么是Copy-on-Write

`Copy-on-Write`顾名思义就是写时复制，是一种并发优化的设计思想。核心原则是**读不加锁，直接读原始数据。写时创建一个副本，先修改副本，等修改完成直接替换原始数据。** 整个过程对读线程透明，读写互不阻塞。

## 优点

1. 读操作零锁竞争：多个读线程完全并发，不会互相阻塞，也不会被写线程阻塞
2. 迭代安全：由于写操作在副本执行，迭代器持有的是旧数组快照
3. 实现简单

## 缺点

1. 写操作开销大：需要复制整个数组（在Java中如此）
2. 内存占用高：写操作期间，新旧两个数组同时存在内存中
3. 数据一致是"最终一致"：在写操作期间，读仍然是旧数据

**PS**：在操作系统领域，Copy-on-Write可以做到按需复制，即不复制主线程的整个内存空间，只复制修改部分的页数据。由于Java运行在JVM上，没有直接操作页表和MMU（内存管理单元）的能力。Copy-on-Write最大的应用领域是在函数式编程，因为函数式编程的基础是不可变性，因此所有修改操作都需要Copy-on-Write来解决。
